# Phase 2: Training CLI - Research

**Researched:** 2026-04-09
**Domain:** RVC training pipeline orchestration from a host Python CLI via subprocess-only, two-venv boundary
**Confidence:** HIGH (all argv shapes verified by reading the pinned RVC commit `7ef19867780cf703841ebafb565a4e47d1ea86ff` directly)

## Summary

Phase 2 delivers `src/train.py` — a typer CLI that orchestrates four RVC training stages (preprocess, F0 extract, feature extract, train) end-to-end by shelling out to scripts inside `rvc/.venv`, with always-on probe-and-skip resume, doctor pre-flight, and structured error output. All argv shapes, filelist formats, and config paths are already fixed by the pinned RVC commit and have been extracted verbatim from source in this document. The architecture mirrors `src/generate.py` exactly: pure arg-builder functions + doctor pre-flight block + thin `Popen` orchestrator, with the twist that subprocesses are **streamed live** to stdout + `rvc/logs/<exp>/train.log` rather than captured, because training runs for hours.

The CONTEXT.md locks almost every architectural decision. Research's job here is **not** to propose alternatives but to (a) document the exact RVC argv for each stage, (b) document the exact filelist format bytes, (c) enumerate the sentinels, and (d) flag the small handful of choices the planner must still make (num_procs defaults, banner wording, preset dict shape).

**Primary recommendation:** Treat this as a transcription exercise. Every argv lines up one-to-one with `rvc/infer-web.py` lines 218–610. The planner should codify the tables in sections 2, 3, 4 directly as Python data structures and write argv-assertion tests against the exact strings recorded here.

## User Constraints (from CONTEXT.md)

### Locked Decisions

**CLI Surface — Presets + Granular Overrides**
- **D-01:** `src/train.py` ships a `--preset` flag with four values: `smoke` / `low` / `balanced` / `high`. Default `balanced`. Preset resolves to a dict of `epochs`, `batch_size`, `save_every`; any explicit flag overrides via `preset_dict.update(explicit_overrides)` after typer parsing.
- **D-02:** Preset values (authoritative):
  - `smoke`: `epochs=1, batch_size=1, save_every=1`
  - `low`: `epochs=200, batch_size=6, save_every=50` (12 GB VRAM floor)
  - `balanced` (DEFAULT): `epochs=300, batch_size=12, save_every=50` (24 GB cards)
  - `high`: `epochs=500, batch_size=40, save_every=50` (H100/A100-80GB)
- **D-03:** `--preset` and explicit overrides are mixable. `--preset high --epochs 800` yields `epochs=800, batch_size=40, save_every=50`.
- **D-04:** Other defaults: `--sample-rate=40000`, `--rvc-version=v2`, `--f0-method=rmvpe`. All explicit CLI flags per TRAIN-01.
- **D-05:** `--resume` flag is **removed from the CLI**. Resume is always-on probe-and-skip (D-08). Deliberate deviation from TRAIN-01's literal flag list; the requirement's spirit (resumable training) is delivered more robustly by D-08. Planner must note deviation in PLAN.md.

**Filelist Generation (TRAIN-05)**
- **D-06:** `_write_filelist(exp_dir: Path, version: str, sample_rate: int) -> Path` is format-equivalent (not byte-exact) to `rvc/infer-web.py:click_train` lines 500–546. Tests assert: pipe-field count, path fields point at real files in `0_gt_wavs/`, `2a_f0/`, `2b-f0nsf/`, `3_feature768/` (v2) / `3_feature256/` (v1), sid=0, one mute-reference row per mute file found under `rvc/logs/mute/`, resulting file non-empty.
- **D-07:** Mute-reference rows are generated by globbing `rvc/logs/mute/0_gt_wavs/*.wav` at helper-invocation time. No hard-coded count. Phase 1's `check_rvc_mute_refs` validates presence as pre-flight.

**Resume + Stage-Skip Probe Semantics**
- **D-08:** Always-on intrinsic probe-and-skip. `N = count of audio files in --dataset-dir matching AUDIO_EXTS from src/preprocess.py`:
  - **Stage 1 skip** iff `count(rvc/logs/<name>/0_gt_wavs/*.wav) >= N`.
  - **Stage 2 skip** iff `count(rvc/logs/<name>/2a_f0/*.npy) >= N` AND `count(rvc/logs/<name>/2b-f0nsf/*.npy) >= N`.
  - **Stage 3 skip** iff `count(rvc/logs/<name>/3_feature768/*.npy) >= N` (v2) or `3_feature256/*.npy` (v1).
  - On mismatch, stage re-runs (RVC scripts overwrite per-file, safe).
  - Log on skip: `"Stage N: skipping — already populated (<count> files)"`. On mismatch: `"Stage N: re-running — expected >= <N> files, found <count>"`.
- **D-09:** Stage 4 resume defers to RVC's built-in `G_*.pth`/`D_*.pth` checkpoint auto-load. **Fast-path:** if `rvc/assets/weights/<name>.pth` exists and is ≥ 1 KiB, skip Stage 4 entirely.
- **D-10:** Same experiment name + same dataset = resume. Same name + different dataset = file-count mismatch re-runs mismatched stages. User responsible for picking distinct `--experiment-name` for clean slate.

**Doctor Pre-flight (TRAIN-06)**
- **D-11:** New checks in `src/doctor.py`:
  - `check_pretrained_v2_weights(sample_rate: int, version: str, if_f0: bool) -> CheckResult` — verifies files under `rvc/assets/pretrained_v2/` (or `pretrained/` for v1) that `train.py` loads as `-pg`/`-pd`.
  - `check_training_dataset_nonempty(dataset_dir: Path) -> CheckResult` — verifies dir exists, is directory, contains ≥1 file matching `AUDIO_EXTS`.
- **D-12:** `check_hubert_base` and `check_rvc_mute_refs` already plumbed in Phase 1's `--training` composition. Phase 2 consumes, does not re-define. If Phase 1 landed as stubs, Phase 2 planner must identify gap and include bodies.
- **D-13:** `src/train.py` calls full training check set via a doctor helper (e.g., `run_training_checks(dataset_dir=..., sample_rate=..., version=..., if_f0=True)`) before any subprocess. On fail: print `rich.Table` + exit 1 with first fix_hint.

**Subprocess Logging + Output Visibility**
- **D-14:** Training subprocesses **streamed live** to terminal AND tee'd to `rvc/logs/<name>/train.log`. Implementation: `subprocess.Popen(..., stdout=PIPE, stderr=STDOUT, bufsize=1, text=True, cwd=RVC_DIR, env=...)`, iterate `popen.stdout` line-by-line writing to both `sys.stdout` and the log file handle. **No** `capture_output=True`.
- **D-15:** On non-zero exit (not 0 or 61 per TRAIN-07), `_tail` reads last **30 lines** of `rvc/logs/<name>/train.log` (not a Python buffer) with pipeline-stage context prefix. Defensive fallback: if log file missing at failure time (rare — crash before any line written), tail the stderr pipe.
- **D-16:** Per-stage logs all append to the **same** `rvc/logs/<name>/train.log`, with a stage-header banner (`"===== Stage N: <name> (started HH:MM:SS) ====="`) before each subprocess.

**Verification + Exit Codes**
- **D-17:** Exit codes (TRAIN-12): 0 success / 1 config-setup (missing weights, missing dataset, failing doctor) / 2 user-input (bad preset, bad sample rate, bad f0 method, bad rvc version) / 3 subprocess runtime (Stage N non-{0,61} exit).
- **D-18:** After Stage 4, cross-check `rvc/assets/weights/<name>.pth` exists and ≥ 1 KiB. If missing despite 0/61 exit: exit 3 with tail + hint `"RVC reported success but no weight file was produced — check train.log for silent failures"`.
- **D-19:** Subprocess env for every stage: `os.environ` copy + `TRANSFORMERS_OFFLINE=1`, `HF_DATASETS_OFFLINE=1`, `LANG=C.UTF-8` (TRAIN-14). Set once in a helper.

**Arg Builders (TRAIN-03, TRAIN-04)**
- **D-20:** Four pure arg-builder functions in `src/train.py`, keyword-only args, return `list[str]`, zero side effects, fully unit-tested without GPU. Authoritative signatures:
  - `build_rvc_preprocess_cmd(*, rvc_python: Path, dataset_dir: Path, sample_rate: int, num_procs: int, exp_name: str) -> list[str]`
  - `build_rvc_extract_f0_cmd(*, rvc_python: Path, exp_name: str, num_procs: int, f0_method: str) -> list[str]`
  - `build_rvc_extract_feature_cmd(*, rvc_python: Path, exp_name: str, version: str, num_procs_per_gpu: int) -> list[str]`
  - `build_rvc_train_cmd(*, rvc_python: Path, exp_name: str, sample_rate: int, version: str, epochs: int, batch_size: int, save_every: int, f0_method: str, pretrained_g: Path, pretrained_d: Path, if_f0: bool, gpus: str) -> list[str]`
- **D-21:** `build_rvc_train_cmd` ALWAYS passes `-pg` and `-pd` explicitly with absolute paths, even though RVC argparse defaults to `""`. Closes STATE.md pitfall: "Missing pretrained weights cause silent random-init training."
- **D-22:** `tests/unit/test_train.py` asserts exact argv for each builder for representative inputs. No GPU required.

**Sample-rate Consistency (TRAIN-13)**
- **D-23:** `--sample-rate` is passed to **both** `build_rvc_preprocess_cmd` (resample target) and `build_rvc_train_cmd` (`-sr`) from the same single variable in `main()`. No intermediate chain. Unit tests cover both with same value.

**Two-Venv Boundary (TRAIN-10)**
- **D-24:** `src/train.py` must NOT contain `import torch`, `import fairseq`, `import faiss`, `import rvc`. All RVC interaction via `subprocess.Popen(..., cwd=RVC_DIR, shell=False)`. Enforce via ruff or a grep check in Phase 2 test suite.

## 2. RVC Training Pipeline Reference

**All argv shapes below are extracted verbatim from `rvc/infer-web.py` at the pinned commit.** They are the canonical contract. `rvc_python = RVC_DIR / ".venv" / "bin" / "python"` and `cwd = RVC_DIR` for every stage. The RVC scripts are invoked with `sys.path.append(now_dir)` where `now_dir = os.getcwd()`, so `cwd=RVC_DIR` is load-bearing.

### Stage 1: Preprocess (`infer/modules/train/preprocess.py`)

**Source:** `rvc/infer-web.py:218-254` (`preprocess_dataset`), script at `rvc/infer/modules/train/preprocess.py` lines 9–15.

Script reads `sys.argv[1..6]`:
1. `inp_root` — dataset directory (absolute path, raw audio files)
2. `sr` — sample rate **in Hz** (integer: 32000, 40000, or 48000; NOT "40k")
3. `n_p` — num processes
4. `exp_dir` — full path `{now_dir}/logs/{exp_name}` (RVC joins here, unlike `train.py` which joins internally)
5. `noparallel` — literal `"True"` or `"False"` string (RVC checks `sys.argv[5] == "True"`)
6. `per` — preprocess_per float (webui default `3.7`; config.py sets `self.preprocess_per = 3.7`)

**Exact argv (CONTEXT.md D-20 signature, with rvc-dir resolution in the thin runner):**
```python
[
    str(rvc_python),                                  # e.g. /abs/rvc/.venv/bin/python
    "infer/modules/train/preprocess.py",              # RELATIVE to cwd=RVC_DIR
    str(dataset_dir.resolve()),                       # inp_root (absolute)
    str(sample_rate),                                 # "40000" not "40k"
    str(num_procs),                                   # e.g. "8"
    str((RVC_DIR / "logs" / exp_name).resolve()),     # exp_dir full path
    "False",                                          # noparallel: we use parallel
    "3.7",                                            # preprocess_per: match webui default
]
```

**Inputs:** `dataset_dir/` containing raw audio files.
**Outputs:**
- `rvc/logs/<exp>/0_gt_wavs/*.wav` — resampled to `<sr>` Hz, amplitude-normalized
- `rvc/logs/<exp>/1_16k_wavs/*.wav` — 16 kHz copies for downstream feature extraction
- `rvc/logs/<exp>/preprocess.log` — log file

**Natural sentinel:** `count(0_gt_wavs/*.wav) >= N` where `N = len(list(dataset_dir.glob(AUDIO_EXTS)))`. Per D-08.

**Dataset-dir contract:** `preprocess.py` uses `sorted(os.listdir(inp_root))` and calls `load_audio(path, self.sr)` (which uses ffmpeg under the hood). It accepts **any** audio format ffmpeg can read — flat directory of `.wav`/`.mp3`/`.flac`/etc. No filename constraints. `src/preprocess.py`'s output (`dataset/processed/*.wav`) is directly compatible.

**GPU?** No. Stage 1 is CPU-only (scipy butter filter + librosa resample). Runs without GPU visible.

### Stage 2: F0 Extraction (`infer/modules/train/extract/extract_f0_print.py` or `extract_f0_rmvpe.py`)

**Source:** `rvc/infer-web.py:258-346` (`extract_f0_feature`, f0-half). Two branches:

**Branch A: `f0method ∈ {"pm", "harvest", "dio", "rmvpe"}`** — script `extract_f0_print.py`.
```python
[
    str(rvc_python),
    "infer/modules/train/extract/extract_f0_print.py",
    str((RVC_DIR / "logs" / exp_name).resolve()),   # exp_dir (sys.argv[1])
    str(num_procs),                                  # n_p (sys.argv[2])
    f0_method,                                       # "rmvpe" | "pm" | "harvest" | "dio" (sys.argv[3])
]
```
Reads `sys.argv[1..3]` (see `rvc/infer/modules/train/extract/extract_f0_print.py:21-35`).

**Branch B: `f0method == "rmvpe_gpu"`** — script `extract_f0_rmvpe.py`, requires GPU partitioning.
```python
[
    str(rvc_python),
    "infer/modules/train/extract/extract_f0_rmvpe.py",
    "1",                                             # leng = number of GPU shards (single-GPU: "1")
    "0",                                             # idx = shard index
    "0",                                             # n_g = GPU device id (single-GPU: "0")
    str((RVC_DIR / "logs" / exp_name).resolve()),    # exp_dir
    "True",                                          # is_half (config.py default on 4090)
]
```
Reads `sys.argv[1..5]` (see `rvc/infer/modules/train/extract/extract_f0_rmvpe.py:11-15`).

**CONTEXT.md D-04 default is `rmvpe`** (not `rmvpe_gpu`), so the v1 default path is Branch A. The planner should implement Branch A first; Branch B is a follow-on that TRAIN-01 implicitly requires via `--f0-method` accepting `rmvpe_gpu`. Both branches must pass argv-assertion tests.

**Outputs:**
- `rvc/logs/<exp>/2a_f0/*.npy` — coarse f0 (integer bins)
- `rvc/logs/<exp>/2b-f0nsf/*.npy` — continuous f0 for NSF
- Appends to `rvc/logs/<exp>/extract_f0_feature.log`

**Natural sentinel:** `count(2a_f0/*.npy) >= N AND count(2b-f0nsf/*.npy) >= N`. Per D-08.

**GPU?** `rmvpe` CPU-or-GPU (loads model on "cuda" by default inside script — see `extract_f0_rmvpe.py:48`). `rmvpe_gpu` requires GPU. `pm`/`harvest`/`dio` are CPU-only. Doctor pre-flight already requires `check_rvc_torch_cuda` so CUDA is guaranteed present regardless.

[
    str(rvc_python),

**Outputs:**
- `rvc/logs/<exp>/3_feature768/*.npy` for `version="v2"` (HuBERT 768-dim)
- `rvc/logs/<exp>/3_feature256/*.npy` for `version="v1"` (HuBERT 256-dim)
- Appends to `rvc/logs/<exp>/extract_f0_feature.log`

**Natural sentinel:** `count(3_feature{768|256}/*.npy) >= N`. Per D-08.

**GPU?** Yes — loads HuBERT on cuda and infers.

### Pre-Stage 4: Filelist + Config Generation

**This is the `_write_filelist` helper (TRAIN-05, D-06/D-07)**, plus a config.json copy that research discovered is **also required** by `train.py` but not mentioned in CONTEXT.md.

**Source:** `rvc/infer-web.py:480-569` (`click_train` lines 480–569).

#### 2.5.1 Filelist format

Location: `rvc/logs/<exp>/filelist.txt`. Shuffled before write.

**Per-dataset-clip row (with f0, `if_f0_3=True`):** 5 fields, pipe-separated:
```
<gt_wavs>/<name>.wav|<feature_dir>/<name>.npy|<f0_dir>/<name>.wav.npy|<f0nsf_dir>/<name>.wav.npy|<spk_id>
```
Where:
- `<gt_wavs>` = absolute path `{now_dir}/logs/{exp}/0_gt_wavs`
- `<feature_dir>` = absolute path `{now_dir}/logs/{exp}/3_feature768` (v2) or `3_feature256` (v1)
- `<f0_dir>` = absolute path `{now_dir}/logs/{exp}/2a_f0`
- `<f0nsf_dir>` = absolute path `{now_dir}/logs/{exp}/2b-f0nsf`
- `<name>` = file stem (join on intersection of names in all four dirs — see `click_train:493-498`)
- `<spk_id>` = `"0"` (single-speaker)

**Per-dataset-clip row (no f0):** 3 fields:
```
<gt_wavs>/<name>.wav|<feature_dir>/<name>.npy|<spk_id>
```

**Mute-reference row (with f0):** RVC webui hardcodes `range(2)` (appends each mute line exactly **twice**). CONTEXT.md D-07 overrides this to "one row per file found under `rvc/logs/mute/0_gt_wavs/`" — verify that `click_train` hardcodes 2 regardless of file count. **Confirmed from reading `infer-web.py:532-537`:** the loop is `for _ in range(2):` and appends the same fixed mute path. D-07 deviates intentionally — the planner should note this in PLAN.md as a format equivalence (the parser accepts it) and D-07 is "format-equivalent, not byte-exact" per D-06.

Mute row template (webui, byte-exact):
```
{now_dir}/logs/mute/0_gt_wavs/mute{sr2}.wav|{now_dir}/logs/mute/3_feature{fea_dim}/mute.npy|{now_dir}/logs/mute/2a_f0/mute.wav.npy|{now_dir}/logs/mute/2b-f0nsf/mute.wav.npy|{spk_id}
```
where `sr2 ∈ {"32k", "40k", "48k"}` and `fea_dim = 768` (v2) or `256` (v1).

**Mute-reference row (no f0):** 3 fields analogous.

**Speaker id (`spk_id5`):** webui default is `0`. D-06 locks `sid=0`.

#### 2.5.2 config.json copy (UNDOCUMENTED in CONTEXT.md — research finding)

`click_train` lines 555–569 also writes `rvc/logs/<exp>/config.json` by copying from `rvc/configs/inuse/{v1|v2}/{sr}.json`. **`train.py` crashes without this file** — it `open(os.path.join(experiment_dir, "config.json"), "r")` at `infer/lib/train/utils.py` line ~376.

Path resolution:
- `version == "v1" OR sr == "40k"` → `configs/v1/<sr>.json` (so v2+40k uses v1 configs — quirk preserved)
- else → `configs/v2/<sr>.json`

The `inuse/` subdir is lazily populated by `Config.load_config_json()` on first webui launch; on a pod that has never launched the webui it may not exist. The planner should **not** assume `configs/inuse/` is populated. Instead `_write_filelist` (or a sibling helper) should read directly from `rvc/configs/{v1,v2}/<sr>.json` and write to `rvc/logs/<exp>/config.json`.

**Recommendation:** Fold this into `_write_filelist` and rename to something broader like `_prepare_experiment_dir(exp_dir, version, sample_rate)` — or document it as a sibling helper `_write_exp_config(exp_dir, version, sample_rate)`. The planner decides; this is in Claude's discretion per CONTEXT.md §"Claude's Discretion." **Raise as an open question** (§9) because CONTEXT.md does not mention this.

Concrete per-stage decision tree, all driven by the same `N = count_dataset_inputs(dataset_dir)` variable computed once in `main()`.

### Resume correctness (TRAIN-09 / success criterion 3)

The ROADMAP requires: "A run aborted after Stage 2 re-invoked completes from Stage 3 onward, not from Stage 1, and produces the same final weight as an uninterrupted run would."

**Analysis of idempotence per stage:**

| Stage | Idempotent? | Why |
|-------|-------------|-----|
| 1 | Yes (per-file overwrite) | `preprocess.py:norm_write` writes `{idx0}_{idx1}.wav` at fixed paths; re-running overwrites. Sorted iteration means indices are deterministic. |
| 2 | Yes | F0 extract writes `.npy` files keyed on input name, overwrites existing. |
| 3 | Yes | Feature extract writes `.npy` files keyed on input name, overwrites existing. |
| 4 | Deterministic-ish | RVC sets `torch.manual_seed(hps.train.seed)` (constant per config) but `torch.backends.cudnn.deterministic = False` in `train.py:39`. Non-deterministic CUDNN + multi-threaded data loader = **same final weight is NOT guaranteed byte-for-byte** even uninterrupted. |

**Interpretation:** The success criterion must mean "produces a final weight" (not "produces a byte-identical weight"). Phase 2's test is "aborted run completes successfully with a valid `.pth` on second invocation," which IS achievable: stages 1-3 are per-file idempotent; stage 4 resumes from RVC's own checkpoint machinery. The test plan should NOT check for byte-equality of two `.pth` files.

**Recommendation to planner:** The verification test for criterion #3 should be: (a) start a run, (b) let stages 1 and 2 complete, (c) SIGKILL the process mid-stage 3, (d) re-invoke the CLI, (e) verify logs show Stage 1 and Stage 2 both skipped, Stage 3 ran from scratch (acceptable — Stage 3 may have partial output; the count probe forces re-run), Stage 4 ran, (f) verify a `.pth` file exists in `rvc/assets/weights/`. Do **not** bit-compare against an uninterrupted baseline.

**Subtle point on stage 3 partial resume:** If SIGKILL lands mid-stage-3 with, say, 50 of 100 `.npy` files written, the probe sees `count=50 < N=100` and re-runs the entire stage 3. `extract_feature_print.py` does not skip already-existing outputs internally — it regenerates. This wastes ~1 minute of GPU time but is correct. **Not worth optimizing in v1.**

## 6. Error Handling & Tailing

### Exit-code mapping (TRAIN-11, TRAIN-12, D-17)

| Failure | Exit code | Action before exit |
|---------|-----------|---------------------|
| Bad preset name, bad sample rate, bad f0 method, bad version, `--rvc-version v1 --sample-rate 32000` combo | **2** | `typer.echo` error to stderr, no log file touched |
| `run_training_checks()` returns any `ok=False` | **1** | Print `rich.Table` of all results, print `Fix the following:` with first fix_hint, exit |
| Dataset dir empty (count == 0) after check_training_dataset_nonempty passes (race) | **1** | Stderr error |
| Subprocess returns non-0 / non-61 exit code for Stages 1-3 | **3** | Print stage context + `_tail_file(train.log, 30)` + optional CUDA OOM hint |
| Subprocess returns non-0 / non-61 exit for Stage 4 | **3** | Same as above |
| Stage 4 exits 0/61 but `rvc/assets/weights/<name>.pth` missing/too small (D-18) | **3** | Print tail + hint: `"RVC reported success but no weight file was produced — check train.log for silent failures"` |
| Any per-stage post-run sentinel probe fails despite exit 0 (e.g., Stage 3 "exits 0 on missing hubert" silent failure) | **3** | Print tail + stage-specific diagnostic |

### Tail helper (D-15)

```python
def _tail_file(path: Path, n: int) -> str:
    """Return last n lines of path, or empty string if path missing.

    Defensive: catches FileNotFoundError + PermissionError + UnicodeDecodeError.
    For large logs (multi-GB), reads from the end via seek() rather than
    loading the full file.
    """
```

The generate.py `_tail(s, n)` operates on an in-memory string. Phase 2's version differs because D-14's streamed tee means we don't hold a string buffer — only the log file. Keep them as two separate helpers. For the rare case where the log file doesn't exist (stage crashed before any output line), fall back to `_tail(stderr_buffer, n)` — but since D-14 uses `stderr=STDOUT` merged with stdout (no separate stderr pipe), in practice the fallback is "log file empty means subprocess crashed at startup, print `'(no output captured before crash)'`."

### `check_pretrained_v2_weights` body sketch

```python
def check_pretrained_v2_weights(
    sample_rate: int, version: str, if_f0: bool
) -> CheckResult:
    sr_str = {32000: "32k", 40000: "40k", 48000: "48k"}.get(sample_rate)
    if sr_str is None:
        return CheckResult(
            name="rvc pretrained weights",
            ok=False,
            detail=f"unsupported sample_rate={sample_rate}",
            fix_hint="Use --sample-rate 32000, 40000, or 48000",
        )
    sub = "pretrained_v2" if version == "v2" else "pretrained"
    prefix = "f0" if if_f0 else ""
    g = RVC_DIR / "assets" / sub / f"{prefix}G{sr_str}.pth"
    d = RVC_DIR / "assets" / sub / f"{prefix}D{sr_str}.pth"
    missing = [p for p in (g, d) if not p.exists()]
    if missing:
        names = ", ".join(p.name for p in missing)
        return CheckResult(
            name="rvc pretrained weights",
            ok=False,
            detail=f"missing: {names}",
            fix_hint=f"Run scripts/setup_rvc.sh to download {names}",
        )
    # Truncation guard (30 MB floor matches Phase 1's f0G40k.pth floor)
    PRETRAINED_MIN_BYTES = 30_000_000
    for p in (g, d):
        if p.stat().st_size < PRETRAINED_MIN_BYTES:
            return CheckResult(
                name="rvc pretrained weights",
                ok=False,
                detail=f"{p.name} is {p.stat().st_size} bytes (expected >= {PRETRAINED_MIN_BYTES})",
                fix_hint="Truncated download. Run ./scripts/setup_rvc.sh --force.",
            )
    return CheckResult(
        name="rvc pretrained weights",
        ok=True,
        detail=f"{sub}/{prefix}{{G,D}}{sr_str}.pth",
    )
```

## 9. Open Questions / Risks

### Open Questions (planner decides)

1. **`config.json` helper location and name.** §2.5.2 found that `train.py` requires `rvc/logs/<exp>/config.json` and CONTEXT.md does not mention it. Options:
   - Fold into `_write_filelist` and rename broader (e.g., `_prepare_experiment_dir`).
   - Separate sibling helper `_write_exp_config(exp_dir, version, sample_rate)`.
   - **Recommendation:** Separate helper — it's independent of the filelist and has a distinct test surface. Call both sequentially in `main()`.
2. **Where `config.json` is copied from.** The webui reads from `rvc/configs/inuse/{v1,v2}/<sr>.json`, which is lazily populated by `Config.load_config_json()` on first webui launch and may not exist on a cold pod. **Recommendation:** Read directly from `rvc/configs/{v1,v2}/<sr>.json` (the tracked source files), which are always present post-clone. Verify by listing `rvc/configs/` during planning.
3. **`num_procs` default.** CONTEXT.md discretion. Webui default from `config.py` uses `os.cpu_count()` indirectly (no cap). `min(os.cpu_count() or 1, 8)` is research's recommendation — matches RVC community defaults and prevents runaway parallelism on 32-core pods.
4. **Handling `--rvc-version v1 --sample-rate 32000` invalid combo.** Webui silently changes sr to 40k (`change_version19:441-443`). Research recommends **rejecting with exit 2** rather than silent correction — explicit is better than magic. Planner confirms.
5. **rmvpe vs rmvpe_gpu default + `--f0-method` validation of `dio`.** CONTEXT.md D-04 locks default to `rmvpe`. `extract_f0_print.py` supports `pm`, `harvest`, `dio`, `rmvpe`; `extract_f0_rmvpe.py` supports only `rmvpe_gpu`. Planner must decide if `dio` is in the valid-methods set (requirements list it as "pm/harvest/rmvpe/rmvpe_gpu"; research finding is that `dio` also works in the script). **Recommendation:** Follow TRAIN-01 literally — only expose `{pm, harvest, rmvpe, rmvpe_gpu}`.
6. **`check_rvc_mute_refs` sufficiency.** Phase 1's body only asserts `rvc/logs/mute/` exists and is non-empty at the top level. For Phase 2 filelist correctness, we actually need `rvc/logs/mute/0_gt_wavs/mute40k.wav` (or matching sample rate). **Recommendation:** Leave Phase 1's check as-is (per D-12) and add a filelist-time assertion in `_write_filelist` that verifies the specific mute file for the requested sample rate exists. If missing, raise with a clear error pointing at `scripts/setup_rvc.sh --force`.

### Risks

- **Risk R1: RVC's `train.py` exits 0 without writing `rvc/assets/weights/<name>.pth`.** Already mitigated by D-18 cross-check. Test this explicitly by injecting a fault on a smoke run and asserting exit 3.
- **Risk R2: `extract_feature_print.py` silent exit 0 on missing hubert.** STATE.md pitfall. Mitigated by (a) Phase 1's `check_hubert_base` pre-flight and (b) Stage 3 post-run sentinel probe. Both must fire.
- **Risk R3: Long-running `Popen` with `stdout=PIPE` deadlock.** Classic Python pitfall: if the pipe buffer fills (~64 KB on Linux) and no one reads, the subprocess blocks. D-14's line-by-line iteration with `bufsize=1, text=True` prevents this — **planner must not forget to actually iterate `popen.stdout`** during the subprocess's lifetime, not after `.wait()`.
- **Risk R4: `iter(popen.stdout)` blocks on partial last line if subprocess crashes without newline.** Use `for line in iter(popen.stdout.readline, '')` + `popen.wait()` at the end. Standard idiom.
- **Risk R5: `MASTER_PORT` binding on pod firewalls.** STATE.md open question: some pod providers block localhost TCP ports. `train.py` uses DDP which binds `MASTER_PORT` (typically 29500). If it hangs, the user sees no output after the Stage 4 banner for minutes. V2-TRAIN-03 tracks this; v1 relies on the user Ctrl-C. Planner should add a brief note in the Stage 4 log banner: "If this stage hangs for >2 minutes with no output, the pod may block localhost TCP — see README troubleshooting." *(README troubleshooting is Phase 5; Phase 2 can at least emit the hint.)*
- **Risk R6: `rvc/configs/inuse/` not populated on a cold pod.** Research finding (§2.5.2). Mitigation: read from `rvc/configs/{v1,v2}/<sr>.json` directly, not from `inuse/`. Verify in Phase 2 integration test.
- **Risk R7: Stage 4 exit code 61 confusion.** TRAIN-07 and D-17 require treating 61 as success. Unit test: `assert TRAIN_SUCCESS_EXIT_CODES == (0, 61)` and `assert is_success(61) is True`.
- **Risk R8: `check_rvc_mute_refs` is a shallow directory-level check.** Does not verify `mute40k.wav` specifically exists. A mute-ref file for the requested sample rate could be missing. Mitigation: add the targeted check in `_write_filelist` (see Open Question #6).

### 10.1 Subprocess streaming pattern (D-14)

```python
# Source: adapted from Python docs + PEP 8 subprocess idioms;
# verified pattern from generate.py's shape (but async tee is new).
import subprocess

def _run_stage_streamed(cmd, *, stage_num, stage_name, log_path, env):
    from datetime import datetime
    banner = STAGE_BANNER.format(
        n=stage_num, name=stage_name, ts=datetime.now().strftime("%H:%M:%S")
    )
    with open(log_path, "a", buffering=1) as logf:
        logf.write("\n" + banner + "\n")
        sys.stdout.write(banner + "\n")
        sys.stdout.flush()

        proc = subprocess.Popen(
            cmd,
            stdout=subprocess.PIPE,
            stderr=subprocess.STDOUT,
            bufsize=1,
            text=True,
            cwd=RVC_DIR,
            env=env,
        )
        assert proc.stdout is not None
        try:
            for line in iter(proc.stdout.readline, ""):
                sys.stdout.write(line)
                sys.stdout.flush()
                logf.write(line)
        finally:
            proc.stdout.close()
            proc.wait()
    return proc.returncode
```

## 12. Don't Hand-Roll

| Problem | Don't Build | Use Instead |
|---------|-------------|-------------|
| Live subprocess output streaming | Custom thread + queue | `subprocess.Popen` + `iter(p.stdout.readline, "")` — stdlib idiom |
| Last-N-lines from a large log file | `file.readlines()[-n:]` (blows up on multi-GB) | Seek-from-end helper (short custom util — no library needed) |
| Preset dict + override merge | Overengineered OO config hierarchy | `{**PRESETS[name], **{k:v for k,v in overrides.items() if v is not None}}` |
| Pretrained path resolution | Repeated string formatting at call sites | One `resolve_pretrained_paths()` helper, tested once |
| RVC config.json copying | Reimplement `Config.load_config_json` | Plain `shutil.copy2(src, dst)` from `rvc/configs/{v1,v2}/<sr>.json` |
| File-count sentinel probe | Recursive glob + complex logic | `len(list(exp_dir.glob("0_gt_wavs/*.wav"))) >= N` — one-liner per stage |

## 13. Common Pitfalls

**P1: Forgetting `cwd=RVC_DIR`.** Every RVC script uses `sys.path.append(os.getcwd())`. Without `cwd=RVC_DIR`, `import infer.lib.train.utils` fails. Mitigation: make `cwd=RVC_DIR` a constant in `_run_stage_streamed`, never let the orchestrator pass a different cwd.

**P2: Passing `-sr 40000` to `train.py`.** `train.py` expects `"40k"`, not the integer. Meanwhile `preprocess.py` expects the integer `40000`. Single-source-of-truth via `SR_STR_MAP` at module level.

**P3: Passing `-e <full-path>` to `train.py`.** `train.py` joins with `./logs/` internally (verified at `infer/lib/train/utils.py:378`). Passing a full path creates `./logs/<full-path>` — double-nested. Builder must pass bare `exp_name`.

**P4: Forgetting to write `config.json` before Stage 4.** `train.py` opens it at init and crashes immediately. Not surfaced as a `-h` error — it's a runtime FileNotFoundError. See §2.5.2.

**P5: Subprocess PIPE deadlock.** D-14 mandates line-by-line iteration. Do NOT `proc.communicate()` after a long wait.

**P6: `if_f0=False` + wrong pretrained files.** If user requests a no-f0 model, the pretrained files must be `G40k.pth`/`D40k.pth` (no `f0` prefix). `resolve_pretrained_paths` must honor `if_f0`.

**P7: Calling `popen.wait()` before draining stdout.** Causes deadlock if stdout buffer fills. Drain first, then `wait()`.

**P8: Hardcoding `num_procs` above host CPU count.** Causes thrashing on small pods. Cap at 8.

## 15. Security Domain

Security enforcement is enabled by default. Phase 2's attack surface is small (no network calls, no secrets, all subprocess calls use shell=False).

### Applicable ASVS Categories

| ASVS Category | Applies | Standard Control |
|---------------|---------|------------------|
| V2 Authentication | no | — |
| V3 Session Management | no | — |
| V4 Access Control | no | — |
| V5 Input Validation | **yes** | typer validates CLI types; explicit whitelist for `--sample-rate`, `--rvc-version`, `--f0-method`, `--preset` |
| V6 Cryptography | no | no crypto used |
| V10 Malicious Code | **yes** | No `shell=True`; subprocess argv always a list; RVC repo pinned to commit |
| V12 Files and Resources | **yes** | Experiment name should be constrained to avoid path traversal |

### Known Threat Patterns for `src/train.py`

| Pattern | STRIDE | Standard Mitigation |
|---------|--------|---------------------|
| Command injection via `--experiment-name "../../etc/passwd"` | Tampering | Validate `exp_name` matches `^[a-zA-Z0-9_-]+$`; reject otherwise with exit 2. RVC writes to `rvc/logs/<name>/` and `rvc/assets/weights/<name>.pth` — a traversal could write outside intended tree. |
| Command injection via `--dataset-dir` with shell metacharacters | Tampering | Already mitigated by `shell=False` + list argv. `Path.resolve()` normalizes but doesn't reject symlinks; accept as user-owned risk. |
| Log injection via RVC stdout containing terminal escape codes | Tampering | Accept — `rich.Console` would render escapes but we use plain `sys.stdout.write`; keep it that way for train.log. |
| Tar-slip-style via training dataset filenames | Tampering | Out of scope — `--dataset-url` is Phase 4. Phase 2 trusts `--dataset-dir` contents. |
| Unintended GPU access on multi-tenant pod | Info disclosure | `-g` default `"0"`; user owns override. Phase 2 does not enumerate other GPUs. |

**Action for planner:** Add `--experiment-name` validator: `re.match(r'^[a-zA-Z0-9_-]{1,64}$', name)`. Reject empty/invalid with exit 2.

## 16. Project Constraints (from CLAUDE.md)

- **Python 3.10 exact.** No 3.11 syntax patterns that don't survive 3.10. `from __future__ import annotations` at top.
- **No `import torch/fairseq/faiss/rvc`** in `src/train.py`. Enforced by unit test §8.4.
- **All subprocess calls: `shell=False`** (list argv, never string). No `subprocess.run("rvc/... ...")`.
- **No new pip packages.** Everything from stdlib + existing deps.
- **No interactive prompts.** Every input via CLI flag or env var. Never call `input()`. Never `typer.prompt()`.
- **Exit codes follow convention:** 0/1/2/3 (D-17). Matches `src/generate.py` and `src/doctor.py`.
- **`Optional[X]` not `X | None`** for typer option defaults (typer 0.12.3 quirk; UP007 ruff rule is ignored project-wide).
- **Keyword-only args after `*`** for all pure arg-builders. House style.
- **Google-style docstrings** with Args/Returns/Raises.
- **Module docstring at top** before `from __future__`, listing exit codes and pipeline overview.
- **Line length 100** (ruff-enforced).
- **Custom exceptions inherit from `RuntimeError`** with bracketed context tags in messages.
- **Rich console for user output**, `typer.echo(..., err=True)` for stderr errors.
- **No broad `except Exception`** — catches must be narrow (named exception types).
- **Re-raise with `from exc`** to preserve tracebacks.
